# MinerU MCP 私有文档问答项目说明

## 项目目标

这个项目实现一个 Claude Desktop + MinerU Open MCP Server 的私有文档问答工作流。用户不需要提前建设 RAG 知识库，也不需要把文档先导入向量数据库，只要在 Claude 里提供本地文档的绝对路径，就可以让 MinerU 按需解析 PDF、Word、PPT 等文件，再由 Claude 基于解析结果回答问题。

## 适用场景

- 临时阅读合同、报告、论文、会议材料、项目文档。
- 小团队验证 MCP 是否适合内部知识访问。
- 对比 MCP、API/SDK、RAG 三种方案的工程成本和使用边界。
- 做个人技术项目、作品集或简历项目。

## 本地使用步骤

### 1. 安装 uv / uvx

如果本机没有 `uvx`，先安装 `uv`：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

然后重新打开终端并检查：

```bash
uvx --version
```

### 2. 选择 Claude Desktop 配置

本项目提供两个模板：

- `config/claude_desktop_config.local.example.json`：Claude Desktop 本地启动 `uvx mineru-open-mcp`。
- `config/claude_desktop_config.remote.example.json`：连接 MinerU 远程 MCP 服务。

macOS 的 Claude Desktop 配置文件通常在：

```text
~/Library/Application Support/Claude/claude_desktop_config.json
```

把其中一个模板里的 `mcpServers.mineru` 合并进去，然后重启 Claude Desktop。

### 3. 准备文档路径

不要只拖文件。部分 MCP 客户端会把拖入的文件复制到临时沙盒目录，导致 MCP Server 找不到原文件。更稳定的方式是直接提供完整路径，例如：

```text
/Users/kongxiangrui/Documents/project-report.pdf
/Users/kongxiangrui/Documents/team-review.pptx
```

也可以用脚本生成 prompt：

```bash
./scripts/make_document_prompt.py ~/Documents/project-report.pdf -q "总结主要风险和下一步行动"
```

### 4. 在 Claude 中提问

可以使用 `prompts/document-qa.md` 的模板：

```text
Use MinerU to parse these private document files:
- /Users/kongxiangrui/Documents/project-report.pdf

Question:
总结主要风险、关键决策和下一步行动。

Instructions:
- Answer only from the parsed document content.
- Cite the document filename and page or section when available.
- If the evidence is missing, say what is missing.
```

## Flash 模式和 token 模式

Flash 模式适合快速试用和小文件解析，一般偏 Markdown-only。token / Precision 模式适合更大的文件、更复杂的表格公式抽取、更稳定的团队使用。

团队采用时建议明确默认策略：

- 试用阶段：Flash 模式。
- 生产或重要文档：token / Precision 模式。

## 如何演示这个项目

1. 准备一个 PDF、一个 DOCX、一个 PPTX。
2. 在 Claude Desktop 中确认 MinerU MCP 工具已连接。
3. 用绝对路径让 Claude 解析文档。
4. 分别问三个问题：
   - 文档的核心结论是什么？
   - 有哪些风险、决策、负责人和截止时间？
   - 哪些信息文档没有提供？
5. 展示 `docs/mcp-vs-api-vs-rag.md`，说明为什么这个项目选择 MCP，而不是先做 RAG。
