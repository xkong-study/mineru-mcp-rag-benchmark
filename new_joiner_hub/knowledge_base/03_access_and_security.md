# Access and Security

## Principles

- Only request the access you need.
- Treat internal docs as sensitive by default.
- Use approved systems for secrets, credentials, and environment variables.

## Common onboarding tasks

- Request repository access.
- Request staging access.
- Confirm MFA is enabled.
- Learn where secrets are stored and how to rotate them.

## Questions that belong here

- How do I request access to a repo?
- What is the process for staging credentials?
- Which docs can be shared externally?
